# MiniType — a small language and an SDD type checker

> **This project is also available on GitHub.** Instead of using the zip archive,
> you can visit the repository and clone the project directly:
>
> **Repository:** https://github.com/Artyom7577/mini-type-checker
>
> ```bash
> git clone https://github.com/Artyom7577/mini-type-checker.git
> cd mini-type-checker
> ./build.sh, ./run.sh, ./test.sh, ./serve.sh
> ```

MiniType is a tiny C-like language; this project implements a **type checker**
for it using a **Syntax-Directed Definition (SDD)** — built on a parser generated by **ANTLR4**.

It supports **8 type forms** : `int`, `double`, `char`,
`bool`, `Array(T)`, `Pointer(T)`, function types `T1 -> T2`, and `struct`
records — plus explicit casts and `void` for statements.

---

## 1. The pipeline

```
source → Lexical analysis → Syntax analysis → Semantic analysis → Code generation
         (lexer/tokens)     (parser/tree)      (TYPE CHECKING)
```

This project implements the first three boxes; **type checking is the semantic
analysis box**, and it is implemented as an SDD:

```
source.mt ──(ANTLR4 lexer)──▶ tokens ──(ANTLR4 parser)──▶ parse tree
          ──(TypeChecker visitor = SDD over the tree)──▶ type errors / success
```

An **SDD** attaches a semantic rule `{ … }` to each grammar production that
computes a synthesized attribute `.type`. In this project, the ANTLR-generated
**visitor** provides exactly one method per production, and that method _is_ the
semantic rule: it returns the node's `Type` (the synthesized `.type`). A single
sentinel `type_error` is produced by any failing rule and propagates upward,
and a symbol table provides `getType`/`addType`.

---

## 2. The MiniType language

A program is a sequence of top-level declarations: **structs**, **functions**,
and **global variables**. Statements live inside function bodies.

```c
struct Point { int x; int y; };          // record type

int   gi = 2 + 3 * 4;                     // global variable (int)

int factorial(int n) {                    // function: type  int -> int
    if (n <= 1) { return 1; }
    return n * factorial(n - 1);          // recursion
}

int main() {
    int    a   = 10;          // int
    double b   = 3.14;        // double
    char   ch  = 'x';         // char
    bool   ok  = a < 20;      // bool (from a comparison)
    int    xs[5];             // Array(int)
    int*   p   = &a;          // Pointer(int)   (& = address-of)
    int    v   = *p;          // dereference    (* = deref)
    Point  pt;  pt.x = 1;     // struct + field access
    int    e   = (int) b;     // explicit cast double -> int
    return factorial(a);
}
```

### Grammar (overview)

The full grammar is `src/minitype/MiniType.g4`. Key parts:

```
program   : topDecl* EOF ;
topDecl   : structDecl | funcDecl | varDecl ;
structDecl: 'struct' ID '{' (type ID arraySuffix? ';')+ '}' ';' ;
funcDecl  : type ID '(' (param (',' param)*)? ')' block ;
varDecl   : type ID arraySuffix? ('=' expr)? ';' ;
type      : ('int'|'double'|'char'|'bool'|'void'|ID) '*'* ;   // base/struct + pointer stars
arraySuffix : '[' INT_LIT ']' ;                               // T id[n]  ⇒ Array(T)

stmt      : varDecl | block
          | 'if' '(' expr ')' stmt ('else' stmt)?
          | 'while' '(' expr ')' stmt
          | 'return' expr? ';'
          | expr '=' expr ';'      // assignment
          | expr ';' ;

expr      : expr '(' args? ')'     # Call      // highest precedence (postfix)
          | expr '[' expr ']'      # Index
          | expr '.' ID            # Field
          | ('!'|'-'|'*'|'&') expr # Unary      // not / negate / deref / address-of
          | '(' baseType ')' expr  # Cast
          | expr ('*'|'/'|'%') expr# MulDiv
          | expr ('+'|'-') expr    # AddSub
          | expr ('<'|'<='|'>'|'>=') expr # Rel
          | expr ('=='|'!=') expr  # Eq
          | expr '&&' expr         # And
          | expr '||' expr         # Or
          | INT_LIT | DOUBLE_LIT | CHAR_LIT | 'true' | 'false' | ID | '(' expr ')' ;
```

Precedence is encoded by **alternative order** (highest first); ANTLR4 resolves
it automatically. Casts target base types only (`(int)e`), which keeps `(int)x`
(a cast) unambiguous from `(x)` (parentheses).

## 3. The type system

`Type` (see `src/minitype/types/`):

| Kind          | Class          | Written as                  | Equality              |
| ------------- | -------------- | --------------------------- | --------------------- |
| base          | `BaseType`     | `int double char bool void` | identity (singletons) |
| array         | `ArrayType`    | `Array(T)`                  | structural            |
| pointer       | `PointerType`  | `Pointer(T)`                | structural            |
| function      | `FunctionType` | `(T1, T2) -> R`             | structural            |
| record/struct | `RecordType`   | `struct Name`               | **by-name**           |
| error         | `ErrorType`    | `type_error`                | —                     |

`Array`, `Pointer` and function types use **structural** equality (two types are
equal iff they have the same shape). Records use **by-name** equality — matching
C and most languages, and making recursive structs (`struct Node { Node* next; }`)
trivial.

## 4. The type rules (the SDD)

Each rule below is one `visitX` method in `TypeChecker.java`. `type_error`
propagates: a rule whose operand is already `type_error` returns `type_error`
**without** a new message, so there is no cascade.

| Construct                            | Rule (synthesized `.type`)                                                         |
| ------------------------------------ | ---------------------------------------------------------------------------------- |
| `T id;` / `T id[n];` / `T* id;`      | `addType(id, T)` / `Array(T)` / `Pointer(T)`; redefinition → error                 |
| `int`/`double`/`char`/`bool` literal | `int` / `double` / `char` / `bool`                                                 |
| `id`                                 | `getType(id)`; if undefined → error, `type_error`                                  |
| `E1 + E2`, `-`, `*`, `/`             | operands same ∧ ∈ {int, double} → that type; else `type_error`                     |
| `E1 % E2`                            | both `int` → `int`; else `type_error`                                              |
| `E1 == E2`, `!=`                     | same type ∧ ∈ {int, double, char, bool\*, Pointer} → `bool` (not arrays/functions) |
| `E1 < E2`, `<=`, `>`, `>=` \*        | same type ∧ ∈ {int, double, char} → `bool`                                         |
| `E1 && E2`, `\|\|`, `!E` \*          | operand(s) `bool` → `bool`                                                         |
| `-E` \* (unary minus)                | `int`/`double` → same                                                              |
| `E1[E2]`                             | `E1 : Array(t)` ∧ `E2 : int` → `t`; else `type_error`                              |
| `*E` (deref)                         | `E : Pointer(t)` → `t`; else `type_error`                                          |
| `&E` (address-of)                    | `E` is lvalue ∧ not an array → `Pointer(E.type)`; else `type_error`                |
| `(int)E` / `(double)E`               | scalar→scalar (int/double/char) → target; else `type_error`                        |
| `f(a1,…,an)`                         | `f : (p1,…,pn) -> r` ∧ each `ai : pi` ∧ arity ok → `r`; else `type_error`          |
| `E.field` \*                         | `E : struct` ∧ field exists → field type; else `type_error`                        |
| `lvalue = E`                         | lvalue type == `E.type` → ok; else error                                           |
| `if (E) S`, `while (E) S`            | `E : bool` ∧ body ok → ok; else error                                              |
| `return E;`                          | `E.type` == the function's return type → ok; else error                            |

**Symbol table** (`SymbolTable.java`) = `getType`/`addType`, with
lexical scopes (global / function / block) and a separate table of named
structs. The program is checked in **two passes** so declarations can be used
before they textually appear (mutual recursion, forward references, recursive
structs).

**Error handling** : every error is reported with `line:column`, and checking **continues**
(it does not stop at the first error).

## 5. Build, run, test

Requirements: a **JDK** (tested on Java 21) and internet access **once** (to
download the ANTLR4 jar into `lib/`).

```bash
./build.sh                          # download ANTLR (first time), generate parser, compile
./run.sh examples/ok_functions.mt   # type-check one file
./run.sh examples/err_calls.mt      # see the type errors
./run.sh examples/ok_arithmetic.mt --tree   # also print the parse tree
./test.sh                           # run all examples and verify ok_*→pass, err_*→fail
```

`run.sh` exits `0` on success, `1` on (syntax or type) errors.

### Web visualizer (optional GUI)

For a visual demo there is a tiny built-in web UI — it uses **only the JDK's
HTTP server** (no extra dependencies, no framework, no build step) and shows, for
any program you type: the **tokens**, the **parse tree** (collapsible), and the
**type-check result / errors** (click an error to jump to that line).

```bash
./serve.sh            # then open http://localhost:8000   (./serve.sh 9000 for another port)
```

It reuses the **same** type checker as the command line: both `Main` (CLI) and
`WebServer` (UI) call `Analyzer.analyze(...)`, which runs the one `TypeChecker`.
So there is no duplicated logic — a fix to a type rule updates both front ends at
once. (`WebServer` adds a JSON endpoint `POST /api/check`; `web/index.html` is the
page that renders the response.)

## 7. Scope & limitations (intentional)

- **Static checking only.** Strong-typing slide
  (`int A[10]; A[11]`), an out-of-bounds index is **not** a type error — it is a
  runtime concern. The checker verifies types, not values.
- **Monomorphic** (no Hindley-Milner inference / type variables). This project
  deliberately uses the simpler, fully-explicit SDD style.
- **Casts** are limited to scalar base types (`int`/`double`/`char`). `(bool)e`
  parses but is always a type error (bool is not in the conversion set).
- Records use **by-name** equivalence. A struct that contains _itself directly_
  (e.g. `struct A { A a; }`) is accepted by the type rules even though it has no
  finite layout — detecting that needs a finite-size/occurs check, which (like
  out-of-bounds). Recursion is meant to go through a pointer (`A* next`).
- A **function name is not assignable and not addressable** (`f = ...` and `&f`
  are rejected) — a function is not a storage location. `void*` _is_ allowed
  (a common idiom).
- **Robustness:** extremely deep nesting (thousands of operators) is reported as
  `input is too deeply nested` rather than crashing.
